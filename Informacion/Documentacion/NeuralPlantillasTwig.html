NeuralPlantillasTwig Clase de Ayuda para el manejo del Motor de Plantillas Twig

NeuralPHP Framework provee una clase que simplifica la instancia del motor de plantillas. http://twig.sensiolabs.org/


NeuralPlantillasTwig('MiApp', 'Cache')


Instancia genera la inclusión de las libreria y el proceso correspondiente.

  1. MiApp: valor por defecto Predeterminado, Selecciona la aplicación donde se tomara las carpetas de origen de las plantillas.
  2. Cache: valor por defecto false, activa el sistema de cache que trae incorporado el motor de plantillas.

								/**
								 * Selección de aplicación
								 */
								$Plantilla = new NeuralPlantillasTwig('MiApp');
								
								/**
								 * Selección de aplicación y activación de Cache
								 */
								$Plantilla = new NeuralPlantillasTwig('MiApp', true);


Filtro('Nombre del Filtro', 'Función Anónima')


El filtro transforma el valor pasado a algo más.

  1. Nombre del Filtro: Nombre que se le dara al filtro correspondiente.
  2. Función Anónima: Es una Función Anónima ó Lambda donde solo tomara un solo y unico parametro.


Ejemplo de Uso en el Controlador

								/**
								 * Un filtro personalizado
							 	 * Para cambiar un texto a Mayusculas
								 */
								$Plantilla->Filtro('Mayusculas', function($Parametro) {
									return mb_strtoupper($Parametro);
								});

Ejemplo de Uso en la Plantilla

								/**
								 * Ejemplo de Uso en la plantilla aplicando
								 * el filtro en una variable de twig
								 */
								{{ Variable|Mayusculas }}
								
								/**
								 * Ejemplo de Uso en la plantilla aplicando
								 * el filtro en un texto en especifico
								 */
								{{ 'un texto que pasara a mayusculas'|Mayusculas }}


Funcion('Nombre del Filtro', 'Función Anónima')


A diferencia del filtro, la función puede pasar varios parametros para ejecutar.

  1. Nombre del Filtro: Nombre que se le dara a la función correspondiente.
  2. Función Anónima: Es una Función Anónima ó Lambda donde tomara los parametros correspondientes.


Ejemplo de Uso en el Controlador

								/**
								 * Una función personalizada
							 	 * Para validar parametros ingresados
								 */
								$Plantilla->Funcion('Escala', function($Parametro_1, $Parametro_2) {
									return ($Parametro_1 >= 5 AND $Parametro_2 <= 10) ? 'Normal' : 'Desestabilizado';
								});

Ejemplo de Uso en la Plantilla

								/**
								 * Ejemplo de Uso en la plantilla aplicando
								 * la función con variables de twig
								 */
								{{ Escala(Variable_1, Variable_2) }}
								
								/**
								 * Ejemplo de Uso en la plantilla aplicando
								 * la función con valores predeterminados
								 */
								{{ Escala(5, 10) }}


Parametro('Nombre de la Variable', 'Valor de la Variable')


Asigna una variable dentro de la plantilla

  1. Nombre de la Variable: Nombre de la variable asignada.
  2. Valor de la Variable: Valor que se le asignara.


Ejemplo de Uso en el Controlador

$Plantilla->Parametro('Titulo', 'NeuralPHP Framework');

Ejemplo de Uso en la Plantilla

								/**
								 * Ejemplo de Uso en la plantilla aplicando
								 * la variable o parametro correspondiente
								 */
								{{ Titulo }}


MostrarPlantilla('Ruta de la Plantilla')


Este metodo genera el proceso de aplicar la plantilla especificada e imprimir el resultado de la validación del proceso del motor de plantillas.

  1. Ruta de la Plantilla: ruta donde se encuentra la plantilla correspondiente dentro de la carpeta de plantillas de la aplicación.

Para Tener En Cuenta

Cuando se especifica dentro de un Modulo, este metodo buscara la plantilla dentro del modulo correspondiente, al no encontrarlo genera un mensaje de error, si la ruta de la plantilla del modulo es igual a la ruta de la carpeta principal de vistas ubicada en la carpeta MVC, se cargara la vista de la carpeta principal.

Se recomienda que las rutas de los modulos sean diferentes o esten contenidas en carpetas que no existan en la carpeta principal de vistas, pero a su vez puede extender e incluir plantillas sin restricción.

Actualmente se pueden utilizar cualquier extensión de texto como puede ser html, htm, php, txt, phtml, ptl y demás formatos de texto.

echo $Plantilla->MostrarPlantilla('Portada/Plantilla.html');

Introducción a Twig

Explicación Tomada de http://gitnacho.github.io/Twig/

En primer lugar, recuerda que el lenguaje de Twig tiene dos construcciones principales:

								{{ }}: Utilizada para imprimir el resultado de la evaluación de la expresión, imprime en la plantilla el resultado de una expresión;
								{% %}: Utilizada para ejecutar instrucciones, se utiliza para ejecutar declaraciones como bucles FOR, estructuras de control IF.

Una plantilla simplemente es un archivo de texto. Esta puede generar cualquier formato basado en texto (HTML, XML, CSV, LaTeX, etc.) No tiene una extensión específica, .html o .xml están muy bien.

Una plantilla contiene variables o expresiones, las cuales se reemplazan por valores cuando se evalúa la plantilla, y las etiquetas, controlan la lógica de la plantilla.


Variables

La aplicación pasa variables a las plantillas para que puedas combinarlas en la plantilla. Las variables pueden tener atributos o elementos en ellas a los cuales puedes acceder también. Cómo se ve una variable, en gran medida, depende de la aplicación que la proporcione.

Puedes utilizar un punto (.) para acceder a los atributos de una variable (métodos o propiedades de un objeto PHP, o elementos de un arreglo PHP), o la así llamada sintaxis de «subíndice» ([]).

								{{ foo.bar }}
								{{ foo['bar'] }}

Cuando el atributo contiene caracteres especiales (como el guión - se debe interpretar como el operador de sustracción), en su lugar usa la función attribute() para acceder al valor de la variable attribute:

								{{ attribute(foo, 'data-foo') }}

Filtros

Los filtros pueden modificar variables. Los filtros están separados de la variable por un símbolo de tubo (|) y pueden tener argumentos opcionales entre paréntesis. Puedes encadenar múltiples filtros. La salida de un filtro se aplica al siguiente.

El siguiente ejemplo elimina todas las etiquetas HTML del name y lo formatea como nombre propio:

								{{ name|striptags|title }}

Funciones

Las funciones se pueden llamar para generar contenido. Las funciones son llamadas por su nombre seguido de paréntesis (()) y pueden tener argumentos.

Por ejemplo, la función range devuelve una lista que contiene una progresión aritmética de números enteros:

								/**
								 * Ejemplo 1
								 */
								{% for i in range(0, 3) %}
									{{ i }},
								{% endfor %}
								
								/**
								 * Ejemplo 2
								 */
								{{ data|convert_encoding('UTF-8', 'iso-2022-jp') }}

Estructuras de Control

Una estructura de control se refiere a todas esas cosas que controlan el flujo de un programa — condicionales (es decir, if/elseif/else), bucles for, así como cosas tales como bloques. Las estructuras de control aparecen dentro de bloques {% ... %}.

Por ejemplo, para mostrar una lista de usuarios provista en una variable llamada users, usa la etiqueta for:

  • {{ Valor.NombreUsuario }}

Comentarios

Para comentar parte de una línea en una plantilla, utiliza la sintaxis de comentario {# ... #}. Esta es útil para depuración o para agregar información para los diseñadores de otra plantilla o para ti mismo:

								{# nota: inhabilitado en la plantilla porque ya no se utiliza
									{% for user in users %}
										...
									{% endfor %}
								#}

Incluyendo Otras Plantillas

La etiqueta include es útil para incluir una plantilla y devolver el contenido producido de esa plantilla a la actual:

{% include 'sidebar.html' %}

De manera predeterminada se pasa el contexto actual a las plantillas incluidas.

El nombre de archivo de la plantilla depende del gestor de plantillas. Por ejemplo, el Twig_Loader_Filesystem te permite acceder a otras plantillas, dando el nombre del archivo. Puedes acceder a plantillas en subdirectorios con una barra inclinada:

{% include "sections/articles/sidebar.html" %}

Herencia en Plantillas

La parte más poderosa de Twig es la herencia entre plantillas. La herencia de plantillas te permite crear un «esqueleto» de plantilla base que contenga todos los elementos comunes de tu sitio y define los bloques que las plantillas descendientes pueden sustituir.

Suena complicado pero es muy básico. Es más fácil entenderlo comenzando con un ejemplo.

Vamos a definir una plantilla base, base.html, la cual define el esqueleto de un documento HTML simple que puedes usar para una sencilla página de dos columnas:

								
{% block Contenido %}{% endblock %}

								

									{% block PiePagina %}
										© Copyright 2014 by tú.
									{% endblock %}

En este ejemplo, las etiquetas block definen los bloques que las plantillas herederas pueden rellenar. Todas las etiquetas block le dicen al motor de plantillas que una plantilla heredera puede sustituir esas porciones de la plantilla.

Una plantilla hija podría tener este aspecto:

								{% extends "base.html" %}
								
								{% block Contenido %}
									
Index

									

										Bienvenido a mi impresionante página.
									

								{% endblock %}

Aquí, la clave es la etiqueta extends. Esta le dice al motor de plantillas que esta plantilla «extiende» otra plantilla. Cuando el sistema de plantillas evalúa esta plantilla, en primer lugar busca la plantilla padre. La etiqueta extends debe ser la primera etiqueta en la plantilla.

Ten en cuenta que debido a que la plantilla heredera no define el bloque PiePagina, en su lugar se utiliza el valor de la plantilla padre.
