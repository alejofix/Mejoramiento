NeuralBDConsultas Clase de Ayuda para Generar Consultas SQL

NeuralPHP Framework provee una clase para generar consultas, esta opción es una ayuda básica para realizar las consultas correspondientes.


Tipo de Conexión

Actualmente utiliza DBAL como conexión de la libreria Doctrine 2


NeuralBDConsultas('MiApp')


Se genera la instancia de la clase NeuralBDConsultas la cual requiere un parametro que es la Aplicación Configurada.
El parametro correspondiente es la aplicación configurada en el archivo de Base de Datos, tambien es posible pasar el objeto de la conexión 

								/**
								 * Parametro de Aplicación Configurada
								 */
								$Consulta = new NeuralBDConsultas('MiApp');
								
								/**
								 * Parametro de la Conexion pasada como variable
								 */
								$Conexion = NeuralConexionDB::DoctrineDBAL('MiApp');
								$Consulta = new NeuralBDConsultas($Conexion);


Tabla('Nombre de la Tabla')


Selecciona el Nombre de la Tabla donde se realizara la consulta, en dado caso que sea necesario una consulta a multiples tablas se debe pasar una matriz con el nombre de las tablas 

								/**
								 * Consulta para una Tabla
								 */
								$Consulta->Tabla('MiTabla');
								
								/**
								 * Consulta para Multiples Tablas
								 */
								$Consulta->Tabla(array('Tabla_1', 'Tabla_2'));


Columnas('Columna')


Selecciona las Columnas de la Tabla que se mostraran en la consulta. 

								/**
								 * Columnas de una Tabla
								 */
								$Consulta->Columnas('Columna, Columna, Columna');
								
								/**
								 * Columnas de Multiples Tablas
								 */
								$Consulta->Columnas('Tabla_1.Columna_1, Tabla_2.Columna_1, Tabla_1.Columna_2');


Condicion("Condiciones")


Asigna la condición ó condiciones, al momento de utilizar multiples condiciones este generara AND 

								/**
								 * Condición Unica
								 */
								$Consulta->Condicion("Columna = 'Valor'");
								
								/**
								 * Multiples Condiciones
							  	 * SELECT * FROM MITABLA WHERE Columna = 'Valor' OR Columna = 'Valor'
								 */
							 	$Consulta->Condicion("Columna = 'Valor' OR Columna = 'Valor'");
							 	
							 	/**
								 * Multiples Condiciones
							  	 * SELECT * FROM MITABLA WHERE Columna_1 = 'Valor' AND Columna_2 = 'Valor'
								 */
							 	$Consulta->Condicion("Columna_1 = 'Valor'");
							 	$Consulta->Condicion("Columna_2 = 'Valor'");


Ordenar('Columna', 'Tipo de Ordenamiento')


Genera el ordenamiento de los datos dependiendo de la columna indicada.

Los posibles valores del Tipo de Ordenamiento:

  1. ASC: Ascendente
  2. DESC: Descendente

								/**
								 * Orden Unico
								 */
								$Consulta->Ordenar('Columna', 'ASC');
								
								/**
								 * Multiples Ordenamientos
								 */
							 	$Consulta->Ordenar('Columna_1', 'ASC');
							 	$Consulta->Ordenar('Columna_2', 'DESC');


Agrupar('Columna')


Genera la agrupación de los datos por la columna seleccionada.
Pude manejarse pasando multiples columnas en una matriz 

								/**
								 * Agrupación Unica
								 */
								$Consulta->Agrupar('Columna');
								
								/**
								 * Multiples Agrupaciones
								 */
							 	$Consulta->Agrupar(array('Columna_1', 'Columna_2'));


InnerJoin('Tabla_2', 'Tabla_1.Columna', 'Tabla_2.Columna')


Genera el tipo de composición de tablas, permite emparejar filas de distintas tablas de forma más eficiente 

$Consulta->InnerJoin('Tabla_2', 'Tabla_1.Columna', 'Tabla_2.Columna');


Ejecutar('Cantidad', 'Query')


Ejecuta el Query correspondiente y regresa una matriz de datos
Los valores que recibe son booleanos:

  1. Cantidad: valor true y regresa la cantidad de datos del resultado del Query
  2. Query: valor true regresa todos los datos solicitados del Query

return $Consulta->Ejecutar(true, true);


Ejemplos de Uso

Es posible manejarla de dos formas:

Forma 1

En esta forma siempre hay que generar el metodo de Ejecutar.

								$Consulta = new NeuralBDConsultas('MiApp');
								$Consulta->Tabla('MiTabla');
								$Consulta->Columnas('Columna, Columna, Columna');
								$Consulta->Condicion("Columna = 'Valor'");
								$Consulta->Ordenar('Columna', 'ASC');
								return $Consulta->Ejecutar(true, true);


Forma 2

En este caso no es necesario el metodo Ejecutar a menos que requiera alguna modificación de los datos que recibira.

								$Consulta = new NeuralBDConsultas('MiApp');
								return $Consulta->Tabla('MiTabla')
												->Columnas('Columna, Columna, Columna')
												->Condicion("Columna = 'Valor'")
												->Ordenar('Columna', 'ASC');
